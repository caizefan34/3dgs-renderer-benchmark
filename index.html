MethodPurposeReporting rule

  • 50K synthetic — low-density scheduling regime
  • 200K synthetic — intermediate workload
  • 400K synthetic — heavy-overlap stress test

Generated-scene results must not be generalized to all trained scenes.

🎬 Standard Camera Paths

  • spiral — 60 cameras, oscillating radius
  • circle — 50 cameras, fixed orbit
  • flythrough — 30 cameras, linear flight
  • random_walk — 30 cameras, jittered orbit

All renderers use identical camera paths for fair FPS comparison

GPU Profiling Insights

The fastest synthetic timing path is gsplat HiGS. It reduces the view-dependent overdraw and tile-list tails observed in Speedy-Splat and dense gsplat. The earlier real-scene drop also appears in dense gsplat, so it is not attributed specifically to HiGS:

OptimizationMechanismObserved trade-off
Hierarchical partitioningCoarse macro-tiles plus fine raster tilesShorter, better-balanced work lists
Persistent packed sceneActivation and workspace setup outside timingInference-only path with fp16 internals
Adaptive tile sizeTile16 below 300K; tile8 at 400K locallyThreshold is workload-specific
SH32 packingTrades decode work for lower bandwidthUseful for high-density tails, slower at 50K

See docs/optimization_analysis.md for measured ablations and the completed TC-GS integration.

Pareto Frontier

No formal Pareto frontier is published yet. The synthetic workloads have no GT quality, and the current real-scene timing is a short smoke test at a different resolution from the quality audit. A frontier will appear only after speed and GT-relative quality are available for the same compatible cohort.

Recommendations

No framework recommendation is published yet. Deterministic recommendations require quality-gated, cohort-compatible speed, quality, memory, and stability results. Synthetic winners are intentionally not promoted as quality-preserving choices.

Quick Start

# Validate the CPU-safe core python -m pip install -r requirements-test.txt python -m unittest discover -s tests -v # After installing CUDA-enabled PyTorch, run gsplat python -m pip install numpy gsplat python src/scripts/generate_scene.py --gaussians 50000 --output data/scene.ply python src/run_benchmark.py --scene data/scene.ply --camera-path circle --renderers gsplat --frames 100 --warmup 30 --repeats 3 # Full report: results/benchmark_report.html
